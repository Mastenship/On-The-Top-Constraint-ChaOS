=== SECTION: ALTERATION_RECORD_SCHEMA ===

ALTERATION_ID: ID_TOKEN (ALT_XX)
ALTERATION_TYPE: ENUM [BIOLOGICAL, CYBERNETIC, MAGICAL, PSYCHOLOGICAL, ENVIRONMENTAL, OTHER]
ALTERATION_NAME: STRING_DESCRIPTIVE
DURATION: ENUM [PERMANENT, TEMPORARY, CONDITIONAL, EPISODIC]
STATUS: ENUM [ACTIVE, DORMANT, REJECTED, INACTIVE, REMOVED]
OBJECTIVE_SYSTEM_EFFECTS: STRING_DESCRIPTIVE
PHYSIOLOGICAL_CHANGES: STRING_DESCRIPTIVE
NEW_ABILITIES: STRING_DESCRIPTIVE
VULNERABILITIES_CONSTRAINTS: STRING_DESCRIPTIVE

CHARACTER_AWARENESS:
AWARENESS_STATUS: ENUM [FULLY_AWARE, PARTIALLY_AWARE, UNAWARE, DENIED]
PERCEIVED_CAUSE: STRING_DESCRIPTIVE
PERCEIVED_EFFECT: STRING_DESCRIPTIVE
SOURCE_OF_INFORMATION: STRING_DESCRIPTIVE

=== SECTION_BREAK ===

=== SECTION: REGISTRY_RULES ===

- [STATE_ALTERATIONS][ALTERATION_REGISTRY] lists all ALTERATION_ID tokens.
- Each listed ALTERATION_ID must have exactly one corresponding [STATE_ALTERATIONS][ALT_XX] record.
- ALTERATION_ID uses prefix ALT_.
- No duplicate ALTERATION_ID.
- New alteration insertion follows strict SET pair contract.

=== SECTION_BREAK ===

=== SECTION: INVARIANTS ===

- STATUS transitions require explicit trigger evidence.
- REJECTED status should drive chronic stress overlay behavior in downstream COTs.
- DORMANT retains reactivation potential.
- REMOVED should trigger cascade reversal proposals where applicable.
