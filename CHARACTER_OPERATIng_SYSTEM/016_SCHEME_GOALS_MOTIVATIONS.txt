=== SECTION: REGISTRY_RULES ===

- [STATE_GOALS_MOTIVATIONS][GOAL_REGISTRY] lists all GOAL_ID tokens.
- Each listed GOAL_ID must have exactly one corresponding [STATE_GOALS_MOTIVATIONS][GOAL_XX] record.
- GOAL_ID uses prefix GOAL_.
- No duplicate GOAL_ID.
- New goal creation follows strict SET pair contract.

=== SECTION_BREAK ===

=== SECTION: INVARIANTS ===

- URGENCY must align with DRIVE_SERVED intensity.
- CONFIDENCE must be plausible against resource/body/skill context.
- PARENT_GOAL_ID must reference existing GOAL_ID when non-empty.
- STATUS transitions require explicit cue/event support.
- High urgency goals should trigger DPF prediction linkage behavior.
