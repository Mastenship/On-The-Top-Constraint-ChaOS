=== SECTION: OPERATION_POLICY ===

[[LLM_INSTRUCTION]]: Dynamics must reference strict operation vocabulary only.
Allowed operation references:
- SET
- TRANSITION
- ADD_MODIFIER
- REMOVE_MODIFIER
- CAP
- VETO

Forbidden operation references:
- CREATE_NEW_RECORD
- UPDATE
- INCREMENT
- DECREMENT

[[LLM_INSTRUCTION]]: Goal creation uses SET pair pattern:
- SET [STATE_GOALS_MOTIVATIONS][GOAL_REGISTRY] append GOAL_XX
- SET [STATE_GOALS_MOTIVATIONS][GOAL_XX] record payload

=== SECTION: DRIVE_ESCALATION ===

ON_PHYSIOLOGICAL_DEPRIVATION:
- Propose TRANSITION toward higher severity in relevant drive field.

ON_SAFETY_THREAT:
- Propose TRANSITION escalation of safety drives.
- Optional CAP or suppressive handling for higher-order drives if required.

ON_ATTACHMENT_EVENT:
- Positive cues may reduce rejection sensitivity.
- Negative cues may increase rejection sensitivity/intimacy pressure.

=== SECTION: GOAL_FORMATION ===

ON_HIGH_URGENCY_DRIVE:
- Form goal via SET pair pattern.
- Initial urgency and confidence must align with drive/resource context.

ON_OPPORTUNITY_OR_OBSTACLE:
- Propose SET/TRANSITION updates to existing goals or instantiate new goal pair when justified.

=== SECTION_BREAK ===

=== SECTION: CONFLICT_RESOLUTION ===

ON_GOAL_CONFLICT:
- Classify conflict type.
- Propose bounded status/urgency adjustments.
- Use ADD_MODIFIER for short-horizon pressure overlays when appropriate.
