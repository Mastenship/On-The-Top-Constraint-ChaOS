=== SECTION: SYSTEM_WIDE_WRITE_POLICY ===

[[LLM_INSTRUCTION]]: All COTs must follow this policy.

POLICY:
- COTs do not write directly into persistent STATE_ modules.
- COTs write intended persistent changes only as proposals into [COGNITIVE_SCRATCHPAD][PROPOSED_UPDATES_REGISTRY].
- COTs may write impulses into [COGNITIVE_SCRATCHPAD][ACTIVE_IMPULSE_REGISTRY] when their template permits.
- COTs append trace entries to [COGNITIVE_SCRATCHPAD][CASCADE_TRACE].
- STATE_GOVERNANCE_AND_COMMIT is the only writer to persistent STATE_* and STATE_TEMPORARY_MODIFIERS.
- All proposals MUST use canonical bracket paths with canonical module roots from CANONICAL_TOKEN_DICTIONARY.
- Legacy roots and alias translations are forbidden.

PROPOSAL_ENTRY_SCHEMA:
PROPOSAL_ID:
SOURCE_COT:
TARGET_MODULE:
TARGET_FIELD:
OPERATION_TYPE: [SET | TRANSITION | ADD_MODIFIER | REMOVE_MODIFIER | CAP | VETO]
PROPOSED_VALUE:
STRENGTH: [0-100]
MECHANISM_TAG:
EPISTEMIC_TAG: [PERCEPTION | INFERENCE | SPECULATION | COMMITTED_FACT | TOOL_REQUEST]
JUSTIFICATION:
NOTES: [Optional]

[[LLM_INSTRUCTION]]: JUSTIFICATION and NOTES fields are subject to translation-before-commit normalization. Write-surface fields remain untouched.

=== SECTION: STRICT_OPERATION_POLICY ===

[[LLM_INSTRUCTION]]: Enforce strict operation vocabulary.

FORBIDDEN_OPS:
- CREATE_NEW_RECORD
- UPDATE
- DECREMENT
- INCREMENT

NORMALIZATION_GUIDANCE_FOR_COT_AUTHORS:
- Record creation => SET on registry + SET on record block
- Decrement/increment ordinal => TRANSITION with ONE_STEP_DOWN / ONE_STEP_UP
- Generic update => SET

=== SECTION: UPDATE_VERIFICATION_CHECKLIST ===

[[LLM_INSTRUCTION]]: After STATE_GOVERNANCE_AND_COMMIT, verify:
- All proposals are COMMITTED, SUPPRESSED, DOWNGRADED, or VETOED with audit entries.
- Every PROPOSAL_ID appears in FINAL_ACTION_LOG with exactly one final action.
- No invalid OPERATION_TYPE values exist.
- No legacy roots appear in proposals, commits, audits, or traces.
- BODY endocrine transitions obey step limits and invariants.
- SKILLS permanent changes only occur with MECHANISM_TAG SKILL_STATE_CHANGE and valid justification.
- MEMORY creations obey per-cycle limit and include canonical SYSTEM_LINKAGE.
- TEMPORARY_MODIFIERS payloads are schema-valid and caps are enforced.
- No orphaned commits exist (every commit has proposal source).
- EPISTEMIC_TAG_VALID_AND_CONSISTENT_WITH_LINKAGE_AND_ROLE.
