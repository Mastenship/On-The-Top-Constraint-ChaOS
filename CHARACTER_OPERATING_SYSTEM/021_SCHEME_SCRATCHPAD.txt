=== SECTION: SCHEME_SCRATCHPAD ===

[KEEP: Purpose & Role]
Immutable structural definition of the volatile COGNITIVE_SCRATCHPAD workspace. This is instantiated per cycle for intermediate calculations, impulse registry, proposal collection, DPF snapshot, selected action output, temporary coordination flags, governance audit logging, normalization audit tracking, domain runtime summaries, and cascade trace accumulation.

Lifecycle invariants:
- INSTANTIATE at cycle start
- POPULATE by COTs
- READ by synthesis/governance
- COMMIT via STATE_GOVERNANCE_AND_COMMIT
- VERIFY via COT_UPDATE_VERIFICATION
- WIPE completely after verification

[KEEP: Relationship to Other Modules]
Informs: All COTs, COT_DECISION_SYNTHESIS, STATE_GOVERNANCE_AND_COMMIT, COT_UPDATE_VERIFICATION.
Is Informed By: Narrative input and SCHEME_ constraints.

- CONFLICT_FLAG: BOOLEAN

=== SECTION: INVARIANTS ===

- All writes occur through COT/system actors only.
- Proposal schema fields are complete and OPERATION_TYPE is strict-set valid.
- Impulses use canonical field casing (uppercase keys).
- Scratchpad is fully wiped post-verification (no carryover).
- Governance audit must provide final-action closure for every received proposal.
- Normalization audit must be populated whenever commit-bound or display-bound normalization is relevant.
- Reasonable per-cycle bounds enforced for proposals and impulses.
