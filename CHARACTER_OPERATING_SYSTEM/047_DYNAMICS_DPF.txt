[KEEP: Purpose & Role]
Immutable mechanics for DPF synthesis dynamics, prediction lifecycle, certainty drift, and recalculation gating.

=== SECTION: OPERATION_POLICY ===

[[LLM_INSTRUCTION]]: Dynamics must reference strict operation vocabulary only.

Allowed operation references:
- SET
- TRANSITION
- ADD_MODIFIER
- REMOVE_MODIFIER
- CAP
- VETO

Forbidden operation references:
- CREATE_NEW_RECORD
- UPDATE
- INCREMENT
- DECREMENT

[[LLM_INSTRUCTION]]: Prediction creation uses SET pair pattern:
- SET [STATE_DPF][PREDICTION_REGISTRY] append PRD_XX
- SET [STATE_DPF][PRD_XX] record payload

=== SECTION_BREAK ===

=== SECTION: PREDICTION_ACCUMULATION ===

ON_MINOR_CONTRADICTION:
- Propose SET:
  TARGET_FIELD: [STATE_DPF][DPF_METRICS][CERTAINTY_LEVEL][VALUE]
  PROPOSED_VALUE: lower lawful integer within bounded range
  JUSTIFICATION: contradiction cue

- If prediction-specific contradiction is identified:
  Propose SET:
    TARGET_FIELD: [STATE_DPF][PRD_XX][CERTAINTY]
    PROPOSED_VALUE: lower lawful integer within bounded range
    JUSTIFICATION: prediction contradiction cue

- If prediction certainty falls below falsification threshold with sufficient cue support:
  Propose SET:
    TARGET_FIELD: [STATE_DPF][PRD_XX][STATE]
    PROPOSED_VALUE: FALSIFIED or RESOLVED_NEGATIVE
    JUSTIFICATION: contradiction resolution cue

- If contradiction load exceeds current-cycle recalculation tolerance:
  queue [COGNITIVE_SCRATCHPAD][TEMPORARY_STATE_FLAGS][DPF_RECALC_QUEUED] = TRUE

ON_RESOLUTION_EVENT:
- If prediction is confirmed:
  Propose SET:
    TARGET_FIELD: [STATE_DPF][PRD_XX][STATE]
    PROPOSED_VALUE: RESOLVED
    JUSTIFICATION: resolution cue

- Certainty reinforcement may be proposed via:
  - SET to a higher lawful bounded integer for [STATE_DPF][PRD_XX][CERTAINTY]
  - SET to a higher lawful bounded integer for [STATE_DPF][DPF_METRICS][CERTAINTY_LEVEL][VALUE]

ON_DRIVE_ACTIVATION_WITH_OBSTACLE_OR_OPPORTUNITY:
- Create prediction via SET pair pattern.

ON_TIME_DRIFT_WITHOUT_SIGNIFICANT_EVENTS:
- Propose bounded SET adjustments to:
  - [STATE_DPF][DPF_METRICS][CERTAINTY_LEVEL][VALUE]
  - [STATE_DPF][DPF_CORE_STATE][EMOTIONAL_VALENCE][VALUE]
  when cue support indicates drift toward current baseline interpretive state.

=== SECTION_BREAK ===

=== SECTION: RECALCULATION_GUARD ===

CYCLE_RECALC_GUARD:
- Only one full DPF recalculation per cycle.
- Additional contradictions queue [COGNITIVE_SCRATCHPAD][TEMPORARY_STATE_FLAGS][DPF_RECALC_QUEUED] = TRUE.

ON_DPF_CONTRAST_EVENT:
- Override pathway may permit multi-field DPF shifts if full proof is provided.

=== SECTION: GOVERNANCE_COMPATIBILITY_NOTE ===

- DPF certainty updates are integer-bounded SET operations unless a lawful ordinal ladder is explicitly defined elsewhere.
- Prediction lifecycle transitions must resolve to explicit STATE enum endpoints.
- Recalculation gating is volatile scratchpad coordination and does not itself constitute persistent DPF mutation.
