=== SECTION: STATE_GOVERNANCE_AND_COMMIT ===

[KEEP: Purpose & Role]
This module is the sole authorized persistent-state writer for the system. It reads proposal packets from [COGNITIVE_SCRATCHPAD][PROPOSED_UPDATES_REGISTRY], validates them against constitutional and SCHEME_ rules, normalizes commit-bound narrative fields into canonical output lexicon, resolves conflicts deterministically, commits lawful updates to persistent STATE_* modules, writes reconciliation results into [COGNITIVE_SCRATCHPAD][STATE_COMMIT_AUDIT], and appends one TRACE_ENTRY to [COGNITIVE_SCRATCHPAD][CASCADE_TRACE].

[KEEP: Relationship to Other Modules]
Informs:
- all persistent STATE_* mutation outcomes
- [COGNITIVE_SCRATCHPAD][STATE_COMMIT_AUDIT]
- COT_UPDATE_VERIFICATION

Is Informed By:
- [COGNITIVE_SCRATCHPAD][PROPOSED_UPDATES_REGISTRY]
- STATE_GOVERNANCE_OVERLAY
- UPDATE_CASCADE_REGISTRY
- CANONICAL_TOKEN_DICTIONARY
- all relevant SCHEME_* modules
- all relevant STATE_* modules

=== SECTION: PROPOSAL_INTAKE_VALIDATION ===

[[LLM_INSTRUCTION]]: Validate every proposal before reconciliation.

REQUIRED_FIELDS:
- PROPOSAL_ID
- SOURCE_COT
- TARGET_MODULE
- TARGET_FIELD
- OPERATION_TYPE
- PROPOSED_VALUE
- STRENGTH
- MECHANISM_TAG
- EPISTEMIC_TAG
- JUSTIFICATION

VALIDATION_CHECKS:
- SOURCE_COT must be canonical and present in UPDATE_CASCADE_REGISTRY
- TARGET_MODULE and TARGET_FIELD must use canonical roots and bracket-path syntax
- OPERATION_TYPE must be one of:
  - SET
  - TRANSITION
  - ADD_MODIFIER
  - REMOVE_MODIFIER
  - CAP
  - VETO
- Forbidden aliases are suppressed:
  - CREATE_NEW_RECORD
  - UPDATE
  - INCREMENT
  - DECREMENT
- EPISTEMIC_TAG must be valid closed-enum member
- STRENGTH must be in range 0-100
- JUSTIFICATION must be non-empty

INVALID_PROPOSALS:
- suppress at intake
- write audit reason code
- never silently drop

=== SECTION: TRANSLATION_BEFORE_COMMIT ===

[[LLM_INSTRUCTION]]: Normalize commit-bound narrative fields before final commit decision.

NORMALIZE:
- JUSTIFICATION
- NOTES
- narrative subfields inside commit-bound payloads where applicable

DO_NOT_ALTER:
- bracket-path tokens
- OPERATION_TYPE
- EPISTEMIC_TAG
- PROPOSAL_ID
- MECHANISM_TAG
- canonical enum values
- registry IDs and record IDs

RULE:
- If normalization would require changing a write-surface token, suppress proposal with:
  TRANSLATION_VIOLATION

SCRATCHPAD_AUDIT_UPDATES:
- Set [COGNITIVE_SCRATCHPAD][TEMPORARY_STATE_FLAGS][CANONICAL_FORM_PENDING] during normalization
- Set [COGNITIVE_SCRATCHPAD][TEMPORARY_STATE_FLAGS][TRANSLATION_VIOLATION_DETECTED] TRUE if any proposal fails normalization
- Write normalization result into [COGNITIVE_SCRATCHPAD][OUTPUT_SURFACE_NORMALIZATION_AUDIT]

=== SECTION: CONFLICT_RESOLUTION_AND_GOVERNANCE ===

[[LLM_INSTRUCTION]]: Resolve conflicts deterministically using constitutional precedence.

PRECEDENCE_ORDER:
1) VETO
2) TRANSITION
3) SET
4) ADD_MODIFIER
5) REMOVE_MODIFIER
6) CAP

TIE_BREAKERS:
1) Higher STRENGTH wins
2) Earlier COT priority index wins from UPDATE_CASCADE_REGISTRY
3) More specific justification wins
4) More conservative change wins
5) Lexical SOURCE_COT ascending
6) Lexical PROPOSAL_ID ascending

RULES:
- VETO cancels lower-precedence proposals on the same target
- SCHEME_ invariants override nominal winners
- If a proposal can be lawfully narrowed, DOWNGRADE rather than suppress
- If no lawful result exists, SUPPRESS

=== SECTION_BREAK ===

=== SECTION: COMMIT_EXECUTION_RULES ===

[[LLM_INSTRUCTION]]: Apply only lawful post-reconciliation proposals.

SET:
- assign explicit lawful value to target field
- registry-backed creation uses strict SET pair pattern:
  - registry append
  - full record payload
- incomplete creation pairs are suppressed

TRANSITION:
- resolve current value against SCHEME_ ordinal order
- ONE_STEP_UP and ONE_STEP_DOWN are valid relative directives
- ONE_STEP_TOWARD_BASELINE and ONE_STEP_TOWARD_NEUTRAL are valid when deterministically inferable from SCHEME_/domain rules
- multi-step change requires full override proof or is downgraded/suppressed

ADD_MODIFIER:
- commit schema-valid modifier creation or refresh only
- enforce registry-record consistency and modifier invariants

REMOVE_MODIFIER:
- remove or expire lawful existing modifier only
- no orphan modifier state may remain

CAP:
- constrain value or effect to lawful bound only
- CAP never expands a change

VETO:
- cancels lower-precedence conflicting proposal(s)
- does not authorize unrelated mutation

=== SECTION_BREAK ===

=== SECTION: SCHEME_INVARIANT_AND_CAP_ENFORCEMENT ===

[[LLM_INSTRUCTION]]: SCHEME_ constitution takes absolute precedence over final commit outcomes.

ENFORCE:
- valid enum values only
- lawful ordinal transitions only
- one-step rules unless override proof is complete
- per-cycle caps from STATE_GOVERNANCE_OVERLAY
- registry-record consistency
- memory creation caps and linkage requirements
- modifier schema validity
- skill invariants including:
  INSTINCT_LEVEL <= MASTERY_LEVEL
- physiological and temporal invariants
- relational and DPF invariants where applicable

IF_VIOLATION_FOUND:
- DOWNGRADE if nearest lawful result is deterministic
- otherwise SUPPRESS
- write deterministic audit reason code
