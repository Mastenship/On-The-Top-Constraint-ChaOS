=== SECTION: VERIFICATION_CHECKLIST ===

[[LLM_INSTRUCTION]]: Verify deterministically in this order.

1) Audit closure:
- Every PROPOSAL_ID appears exactly once with FINAL_ACTION
- No silent drops

2) Operation validity:
- OPERATION_TYPE values are only:
  SET | TRANSITION | ADD_MODIFIER | REMOVE_MODIFIER | CAP | VETO
- No forbidden ops appear

3) Canonical path integrity:
- No legacy roots in proposals, commits, audits, traces
- TARGET_FIELD paths are canonical and scheme-valid

4) Ordinal/enum validity:
- ONE_STEP rules enforced unless override proof valid
- Enum values legal per SCHEME_

5) Invariants/caps:
- Skills invariants
- Memory creation caps/linkage requirements
- Modifier schema + orphan checks
- Registry-record pair consistency

6) Commit provenance:
- Every commit traces back to proposal source

7) EPISTEMIC_TAG_VALID_AND_CONSISTENT_WITH_LINKAGE_AND_ROLE
- Every proposal carries valid EPISTEMIC_TAG from closed enum
- COMMITTED_FACT only when supported by verified evidence linkage
- Consistency with JUSTIFICATION, MECHANISM_TAG, SYSTEM_LINKAGE, and role bounds
- Invalid/missing/mismatched tags → VIOLATION

If violation found:
- classify as VIOLATION or WARNING
- include deterministic reason code
