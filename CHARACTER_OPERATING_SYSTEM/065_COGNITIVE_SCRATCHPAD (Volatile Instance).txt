=== SECTION: COGNITIVE_SCRATCHPAD ===

[KEEP: Purpose & Role]
Volatile runtime workspace for BIG ChaOS. This container stores all non-persistent cycle artifacts required for proposal staging, impulse arbitration, DPF snapshotting, selected action output, temporary coordination flags, governance audit logging, and cascade trace accumulation. It is instantiated fresh each cycle, populated during cascade execution, read by synthesis/governance stages, verified post-commit, and wiped completely after verification.

[KEEP: Relationship to Other Modules]
Informs:
- All COT modules
- STATE_GOVERNANCE_AND_COMMIT
- COT_DECISION_SYNTHESIS
- COT_COMMUNICATION_PROCESSING
- COT_UPDATE_VERIFICATION

Is Informed By:
- UPDATE_CASCADE_REGISTRY
- STATE_GOVERNANCE_OVERLAY
- SCHEME_SCRATCHPAD
- All active COT outputs during the current cycle

=== SECTION_BREAK ===

=== SECTION: WRITE_POLICY ===

[[LLM_INSTRUCTION]]: This file is the only authorized volatile write surface for normal cycle processing.

RULES:
- COTs may write only to explicitly permitted scratchpad sections.
- Scratchpad contents are temporary and non-persistent unless committed through STATE_GOVERNANCE_AND_COMMIT.
- Scratchpad artifacts must never be treated as substitutes for STATE_* modules.
- Canonical tokens, IDs, bracket-paths, OPERATION_TYPE values, EPISTEMIC_TAG values, MECHANISM_TAG values, and registry identifiers remain untranslated write-surface elements.
- Internal deliberation may occur in any language or mixed style.
- Before any commit decision, commit-bound narrative artifacts must be normalized according to TRANSLATION_BEFORE_COMMIT policy.
- User-visible artifacts emitted from this scratchpad must be normalized before display.
- Scratchpad is fully wiped after COT_UPDATE_VERIFICATION.
